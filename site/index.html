Getting Started

Up and running in minutes

You'll need an OpenAI API key (for embeddings) and an Anthropic API key (for sync & research).

1

Install

Install globally from npm:

npm install -g @mishkinf/lore
2

Setup

Run the guided wizard to configure API keys and log in:

lore setup

You'll enter your OpenAI and Anthropic API keys, then sign in with your email.

3

Add Sources

Point Lore at a folder of documents:

lore sync add

You'll be prompted for a name, path, file pattern, and project. Lore watches these folders for new files.

4

Sync & Search

Index your documents and start searching:

lore sync
lore search "user pain points"
lore research "What auth approach should we use?"
lore browse
MCP Integration

Use with Claude Desktop & Claude Code

Add Lore as an MCP server to give your AI assistant access to your full knowledge base. If you've already run lore setup, your API keys are stored in your config — you only need to set them here if the MCP host doesn't inherit your shell environment.

Claude Code

Add to your project's .mcp.json or ~/.claude.json (global): 

{
  "mcpServers": {
    "lore": {
      "command": "lore",
      "args": ["mcp"]
    }
  }
}

Claude Desktop

Go to Settings → Developer → Edit Config. Claude Desktop doesn't inherit your shell, so include API keys: 

{
  "mcpServers": {
    "lore": {
      "command": "lore",
      "args": ["mcp"],
      "env": {
        "OPENAI_API_KEY": "your-key",
        "ANTHROPIC_API_KEY": "your-key"
      }
    }
  }
}

Cursor / Windsurf / Other MCP Clients

Any MCP-compatible client can connect to Lore. Use this standard config and add it wherever your client expects MCP server definitions: 

{
  "command": "lore",
  "args": ["mcp"],
  "env": {
    "OPENAI_API_KEY": "your-key",
    "ANTHROPIC_API_KEY": "your-key"
  }
}

FAQ

Common questions

What's the difference between Lore and a memory system?

Memory systems store processed facts — you lose the original source. Lore preserves your complete documents and provides citations back to the original content. When an AI uses Lore, it can quote exactly what was said, by whom, and when.

What does sync cost?

Discovery (Phase 1) is free — it only computes file hashes and checks against Supabase. Processing (Phase 2) only runs for new files and uses Claude for metadata extraction plus OpenAI for embeddings. Re-syncing already-indexed files costs nothing.

Can multiple people share a knowledge base?

Lore uses Supabase Auth with Row Level Security — each user's data is isolated. Team sharing features are planned for a future release.

What file formats are supported?

Markdown, JSONL, JSON, plain text, CSV, HTML, XML, PDF, and images (JPEG, PNG, GIF, WebP). Claude automatically extracts text and metadata from all formats.

Do I need to set up any infrastructure?

No. Lore's backend is fully hosted — just install, log in, and go. You only need to provide your own OpenAI and Anthropic API keys.

How does multi-machine sync work?

Each file gets a SHA256 content hash. Same content = same hash, regardless of file path or machine. If you sync the same file from two machines, only the first is processed. Raw documents are also git-synced through your data directory.
